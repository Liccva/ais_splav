# АИС «Сплав»

Автоматизированная информационная система **«Сплав»** — это веб-приложение для хранения данных о металлических сплавах и прогнозирования их предела прочности с помощью методов машинного обучения.  
Проект ориентирован на научных сотрудников и инженеров, которые разрабатывают и анализируют новые сплавы и хотят сократить количество дорогостоящих физических экспериментов.

## Что это?

- Веб-система для учета химического состава сплавов и их физических свойств с централизованным хранилищем данных на базе MariaDB.  
- ML-сервис, использующий модели градиентного бустинга и случайного леса для прогноза предела прочности по заданному составу сплава.  
- Клиент-серверное приложение на стеке React + FastAPI, доступное через браузер.

## Зачем это?

- Сокращение числа натурных экспериментов и связанных с ними затрат за счет предварительного ML-прогнозирования свойств сплавов.  
- Ускорение НИОКР: исследователь может быстро перебрать варианты составов и отобрать перспективные для дальнейших испытаний.  
- Централизация знаний: единая база сплавов, прогнозов, патентов и отчетов, доступная для исследовательской команды.

## Основные возможности

- **Управление пользователями**  
  - Регистрация и аутентификация пользователей (гость, научный сотрудник, администратор).  
  - CRUD-операции над учетными записями и ролями (для администратора).

- **Работа со сплавами и патентами**  
  - Ввод и редактирование данных о химическом составе сплавов и их свойствах.  
  - Просмотр каталога сплавов, связанных патентов и истории экспериментов/прогнозов.

- **ML-прогнозирование**  
  - Ввод состава сплава (концентрации элементов, тип сплава и технологические параметры).  
  - Запуск моделей машинного обучения для прогноза предела прочности и сохранение результатов в БД.  

- **Отчетность и аналитика**  
  - Формирование отчетов по результатам прогнозов и активности пользователей за период.  
  - Экспорт отчетов и результатов прогнозов в файл.

## Как использовать?

1. **Регистрация и вход**  
   - Перейти на веб-интерфейс системы в браузере.  
   - Зарегистрировать аккаунт или войти под существующей учетной записью (гость имеет ограниченный доступ только к просмотру открытых данных).

2. **Работа с данными о сплавах**  
   - Научный сотрудник создает записи о своих сплавах: указывает химический состав, тип сплава и сопутствующую информацию.  
   - При необходимости редактирует или дополняет ранее введенные данные; администратор может управлять всеми записями.

3. **Прогнозирование предела прочности**  
   - На странице прогноза ввести концентрации элементов и параметры интересующего сплава.  
   - Запустить расчет, дождаться выполнения и просмотреть предсказанный предел прочности, а также степень достоверности прогноза.  
   - Сохранить результат в базе и, при необходимости, включить его в отчет.

4. **Отчеты и администрирование**  
   - Сформировать отчет за выбранный период по проведенным прогнозам и используемым сплавам.  
   - Администратор управляет пользователями, ролями, моделями и контролирует корректность данных.

## Установка и запуск

> Ниже приведен примерный сценарий развёртывания для учебного или тестового окружения.

### Предварительные требования

- Сервер или VM с Linux Debian (≥ 4 CPU, 16 GB RAM, 100 GB HDD/SSD).  
- Установленные:  
  - Python 3.x и pip.  
  - Node.js (для сборки фронтенда React).  
  - MariaDB сервер.  
  - Git для получения исходного кода.

### Backend (FastAPI + ML-сервисы)

1. Клонировать репозиторий проекта и перейти в директорию backend.  
2. Создать и активировать виртуальное окружение Python, установить зависимости (FastAPI, Uvicorn, библиотеки для ML-моделей и работы с MariaDB).  
3. Настроить подключение к MariaDB (строка подключения, пользователь, пароль, имя БД).  
4. Выполнить миграции/скрипты создания схемы БД.  
5. Запустить сервер приложений (например, `uvicorn main:app --host 0.0.0.0 --port 8000`).

### Frontend (React)

1. Перейти в директорию frontend.  
2. Установить зависимости командой `npm install` или `yarn`.  
3. Указать адрес backend-API (FastAPI) в конфигурации фронтенда.  
4. Запустить dev-сервер (`npm start`) или собрать production-сборку (`npm run build`) и развернуть её на выбранном веб-сервере (Nginx и др.).

### База данных

- Развернуть MariaDB, создать отдельную БД для АИС «Сплав» и пользователя с ограниченными правами.  
- Импортировать стартовую схему и, при наличии, начальные справочники (химические элементы и т.п.).

После запуска backend-сервиса и фронтенда система будет доступна по адресу, заданному веб-сервером, и готова к использованию для ввода сплавов, запуска прогнозов и формирования отчетов.

## Используемые технологии

- **Frontend**  
  - React для построения одностраничного веб-интерфейса, форм ввода данных и визуализации результатов прогнозов и отчетов.

- **Backend**  
  - Python + FastAPI для реализации REST-API, авторизации, управления данными о сплавах и интеграции с ML-моделями.  
  - ML-модели: Random Forest, XGBoost для прогнозирования предела прочности сплавов.

- **База данных и инфраструктура**  
  - MariaDB как основная реляционная СУБД для хранения данных о сплавах, пользователях, патентах и результатах расчетов.  
  - ОС Linux Debian для серверной части; Git для контроля версий и совместной разработки.

Проект АИС «Сплав» демонстрирует практическое применение машинного обучения в материаловедении и может быть интересен как учебный, исследовательский или демонстрационный кейс для портфолио.
